Async and Try/Catch

When the logic of an async function is wrapped in Try/catch blocks and one of the awaited Promises rejects, Try/catch treats the rejected Promise:
SOLUTION: In the same way as Try/Catch blocks treat errors thrown by JavaScript, meaning the `catch` statement will run

try {
  console.log("A");
  try {
    console.log("B");
    throw new Error("C");
    console.log("D");
  } catch(error) {
    console.log("E");
  } finally {

Async Await

Select the benefits of using Async/await syntax from the list below.
SOLUTION:
  • Allows us to select Promises to be treated as blocking actions with the await word
  • Allows the developer to use the results of many Promises and be confident that the Promises have resolved

QUESTION:

Explain when you would use Async/await in JavaScript.

SOLUTION:

What will the following code output?

const p1 = new Promise((resolve, reject) => {
    console.log('A');
    resolve(['B', 'C']);
})
const p2 = new Promise((resolve, reject) => {
    resolve(['D']);
})
const test = async () => {
  try {
    const d = await p1;
    const d2 = await p2;
    console.log(d.join(" "), d2.join(" "));
  } catch(err) {
    console.log(err)
  };
};

test();
SOLUTION: A, B, C, D

async/await

What would the following code output?

const p1 = new Promise((resolve, reject) => {
    console.log('A');
    resolve('B');
})
const p2 = new Promise((resolve, reject) => {
    resolve(['D']);
});
const test = async () => {
  try {
    const d = await p1.then(data => console.log(data));
    const d2 = await p2;
    console.log(d, d2.join(""));
  } catch(err) {
    console.log(err);
  };
};
SOLUTION: A, B, Undefined, D
